DevOps Agent — 运维智能体

面向服务器运维场景的智能 Agent，通过日志解析、系统指标采集和工具调用能力，实现故障分析、状态监控和自动化运维。
基于 Nebula 记忆引擎，持久化故障记录和解决方案，持续提升问题定位效率。

它解决什么问题？

运维人员每天都在经历的场景：

运维: "服务器 CPU 突然飙升，帮我看看怎么回事"
      ↓ 手动 SSH 登录 → top → df → tail logs → grep ERROR → 翻聊天记录...
      ↓ 半小时过去了，原因还没找到
      ↓ 上次明明处理过类似问题，但不记得怎么解决的了

有了 DevOps Agent 之后：

运维: "服务器 CPU 突然飙升，帮我看看怎么回事"
Agent: "正在采集系统指标... CPU 95%, 内存 78%, 磁盘 62%。"
Agent: "检测到进程 java(PID:12345) CPU 占用 87%。"
Agent: "搜索历史故障... 发现相似案例: 3天前同一服务因 GC 频繁导致 CPU 飙升。"
Agent: "建议: 1) 检查 JVM GC 日志 2) 调整堆大小 3) 上次方案: 增加 -Xmx 到 8G"
      ↓ 30 秒定位问题 + 历史方案直接可用

快速开始

启动服务

cd "DevOps Agent"
go run ./cmd/devops-server --port 8740

# 浏览器打开 http://localhost:8740/web/

API 示例

# 获取系统指标
curl http://localhost:8740/api/v1/devops/metrics

# 执行诊断
curl -X POST http://localhost:8740/api/v1/devops/diagnose \
  -H "Content-Type: application/json" \
  -d '{"query":"CPU 使用率过高，帮我排查","server_name":"prod-01"}'

# 搜索历史故障
curl -X POST http://localhost:8740/api/v1/devops/faults/search \
  -H "Content-Type: application/json" \
  -d '{"query":"CPU 飙升","limit":5}'
