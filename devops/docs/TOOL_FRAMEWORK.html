DevOps Agent — Tool Calling 框架文档

概述

Tool Calling 框架是 DevOps Agent 的核心子系统，实现了系统命令、日志查询、服务管理等运维工具的注册、发现、推荐和执行。设计参考 OpenAI Function Calling / Anthropic Tool Use 接口风格，完全自研，零外部依赖。

核心概念

ToolDef — 工具定义

type ToolDef struct {
    Name           string       // 工具名称（唯一标识）
    Description    string       // 功能描述
    Category       ToolCategory // system/log/service/network/metrics
    Parameters     []ToolParam  // 参数列表
    Examples       []string     // 使用示例
    RiskLevel      string       // low/medium/high/critical
    RequireConfirm bool         // 是否需要用户确认
}

ToolRegistry — 注册中心

type ToolRegistry struct {
    tools map[string]*ToolEntry  // name → {Def, Func}
}
// 核心方法: Register / Execute / ListDefs / ListByCategory

ToolFunc — 执行函数

type ToolFunc func(args map[string]any) (*ToolResult, error)

工具分类

分类常量工具数说明

工具推荐机制

基于关键词匹配评分：每个工具预置一组关键词，用户查询中匹配到则加分（每个+20分，最高80分）。

使用方法

定义 + 注册工具

// 1. 定义
toolDef := agent.ToolDef{
    Name: "my_tool", Description: "我的工具",
    Category: agent.CatSystem,
    Parameters: []agent.ToolParam{
        {Name: "target", Type: "string", Required: true},
    },
}

// 2. 实现
func myToolFunc(args map[string]any) (*agent.ToolResult, error) {
    // 执行运维操作...
    return &agent.ToolResult{Success: true, Output: "ok"}, nil
}

// 3. 注册
agent.Registry().Register(toolDef, myToolFunc)

API 调用

curl -X POST http://localhost:8740/api/v1/devops/tools/execute \
  -H "Content-Type: application/json" \
  -d '{"name":"my_tool","args":{"target":"server-01"}}'

工具调用时序

Client → POST /tools/execute → API Handler
    → ToolRegistry.Execute(ToolCall)
    → ToolRegistry.Get(name) → ToolEntry
    → fn(args) → 执行操作
    → ToolResult → JSON Response

安全设计

机制说明
风险等级low/medium/high/critical 四级
RequireConfirm高风险操作标记需确认
超时控制ExecuteWithContext() 支持 context 超时
错误处理执行失败返回 ToolResult{Success:false}，不 panic

扩展指南

  1. 在 tools/ 下新建文件实现 ToolFunc
  2. 在对应的 Register*Tools 函数中注册
  3. 在主入口调用注册函数
  4. 在 agent/agent.go 的 keywords map 中添加关键词
