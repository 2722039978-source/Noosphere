DevOps Agent — 系统指标采集模块文档

概述

系统指标采集模块负责从操作系统层面采集 CPU、内存、磁盘、进程 等运维关键指标，同时支持 Windows 和 Linux 平台。

架构设计

Collector (接口)
    ▼
SystemCollector (实现)
    ├── CollectCPU()      → Win: Win32_Processor / Linux: /proc/stat
    ├── CollectMemory()   → Win: Win32_OperatingSystem / Linux: /proc/meminfo
    ├── CollectDisk()     → Win: Win32_LogicalDisk / Linux: df -BG
    ├── CollectProcesses() → Win: Get-Process / Linux: ps aux
    └── Collect()         → 聚合以上 + 5秒缓存

性能优化

优化实现
缓存机制Collect() 5秒缓存，避免频繁系统调用
并发安全sync.RWMutex 保护缓存读写
按需采集单独采集 CPU/内存/磁盘时跳过缓存

使用示例

// Go 代码
collector := metrics.NewSystemCollector()
snapshot, _ := collector.Collect()
fmt.Printf("CPU: %.1f%%, Mem: %.1f%%", snapshot.CPU.UsagePercent, snapshot.Memory.UsagePercent)

// API 调用
curl http://localhost:8740/api/v1/devops/metrics
// → {"cpu":{"usage_percent":45.2}, "memory":{"usage_percent":57.8}, ...}
