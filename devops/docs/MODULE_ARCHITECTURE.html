模块详情

1. agent/ — 核心引擎

职责：Agent 生命周期管理、任务编排、工具推荐

文件功能
types.go所有核心类型定义：AgentState、ToolDef、AgentTask、AgentResponse 等
agent.goDevOpsAgent 主体：Diagnose() 诊断流程、工具引导推荐
tool_registry.goToolRegistry：工具注册、查找、执行、按分类列出

2. tools/ — 运维工具实现

文件工具平台
system.goexec_command, get_system_info, list_processes, check_portWin+Linux
log.gosearch_logs, parse_error_log, tail_logWin (PowerShell)
service.gocheck_service, restart_service, list_servicesWin+Linux

3. metrics/ — 系统指标采集

接口方法功能Win 实现Linux 实现
CollectCPU()CPU 使用率Win32_Processor/proc/stat
CollectMemory()内存指标Win32_OperatingSystem/proc/meminfo
CollectDisk()磁盘指标Win32_LogicalDiskdf -BG
CollectProcesses()TOP N 进程Get-Processps aux

4. analyzer/ — 日志异常分析

规则引擎 + LLM 双层分析。预置 12 条异常规则包括：OOM_Killer、DiskFull、ConnectionRefused、ServiceCrash、DBConnectionFail 等。

5. memory/ — Nebula 记忆集成

方法功能
SaveFault()双写：情景记忆(JSON) + 语义记忆(解决方案)
SearchSimilarFaults()混合检索历史故障
SearchSolutions()检索语义记忆中的解决方案
GetFaultStats()故障统计信息

6. api/ — REST API 端点

端点方法功能
/api/v1/devops/statusGETAgent 状态 + 指标
/api/v1/devops/metricsGET系统指标快照
/api/v1/devops/diagnosePOST提交诊断任务
/api/v1/devops/toolsGET列出所有工具
/api/v1/devops/tools/executePOST执行工具调用
/api/v1/devops/logs/analyzePOST日志分析
/api/v1/devops/faultsGET/POST故障记录 CRUD
/api/v1/devops/faults/searchPOST搜索历史故障

数据流

诊断流程

用户输入 "CPU 使用率过高"
    │ 1. 关键词分析 → 匹配工具
    │ 2. 采集系统指标 (CPU/Mem/Disk)
    │ 3. 搜索历史故障 (向量+关键词)
    │ 4. 生成诊断报告
    ▼
AgentResponse { Diagnosis, MetricsSnapshot, RelatedFaults }

工具调用流程

POST /tools/execute {name:"list_processes", args:{...}}
    → ToolRegistry.Get("list_processes")
    → listProcessesTool(args)
    → ToolResult {Success, Output, Duration}
