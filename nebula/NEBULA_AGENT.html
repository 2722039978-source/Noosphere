◈ Nebula Agent（星云智能体）

让你的 AI 工具真正"懂"你的项目。
扫描代码库 → 学习编码风格 → 自动注入到 AI 对话中。
零外部依赖，一个 Go 二进制文件搞定。

它解决什么问题？

❌ 你每天都在经历的痛点：

你打开 ChatGPT/DeepSeek/Claude：

你: "帮我给项目加一个认证中间件"
AI: "你的项目用什么语言？什么框架？有什么编码规范？..."

你: "用 Go，Gin 框架，JWT 认证，统一返回 {code, data, msg} 格式"
AI: "好的..." (生成了不符合你风格的代码)

→ 每次对话都要重复解释
→ 生成的代码风格总是不对
→ AI 根本不了解你的项目
✅ 有了 Nebula 之后：

你: "帮我给项目加一个认证中间件"
AI: "好的。根据你的项目(Gin+JWT，PascalCase/camelCase，显式 error 返回，统一 {code,data,msg} 格式)，这是代码: ..."

→ AI 已经"知道"你的项目了
→ 代码风格自动匹配
→ 0 次重复解释

怎么做到的？

你的代码库 (Go/Python/TypeScript/...)
↓ ① nebula ingest --dir . (扫描项目)

Nebula 记忆引擎 —— 学到了什么？
语言: Go(85%) | 框架: Gin, GORM, JWT | 命名: PascalCase/camelCase
错误处理: 显式 error 返回 | API 格式: {code, data, message}
⚠ 注意事项: JWT secret 在环境变量，不要硬编码

↓ ② 自动检索 + 拼接 System Prompt

AI 收到的完整指令：
"## 项目上下文 (Nebula 注入)
技术栈: Go, Gin, GORM, JWT
编码风格: PascalCase/camelCase，显式error返回
⚠ JWT secret 在环境变量中
请遵循以上规范生成代码"

↓ ③ 发送给 AI

你的 AI 工具 (ChatGPT / DeepSeek / Claude / ...)
↓
AI 返回符合你风格的代码

三种使用方式

方式 1：浏览器直接对话（最简单）

go run ./cmd/nebula-server --memory --port 8730
# 打开 http://localhost:8730 → 右侧 AI 编程助手

内置 DeepSeek V4 Pro，自动注入项目记忆，打开即用。

方式 2：生成上下文 → 粘贴到任何 AI 工具

go run ./cmd/nebula-code context "添加认证中间件" --lang go | clip
# 打开 ChatGPT → 粘贴 → 直接提问

适合已经习惯用网页版 AI 工具的用户。

方式 3：嵌入你的代码中

import "github.com/nebula-agent/nebula/engine"

eng, _ := engine.Open(engine.MemoryOptions())
sess := eng.Session("my-project")
sess.RememberEpisodic("用户喜欢 Python", 0.8, []string{"preference"})
results, _ := sess.Reminisce(&manager.SearchOptions{
    Query: "编程语言", TopK: 5,
})

技术栈

模块技术为什么
存储引擎自研 LSM Tree (WAL+SkipList+SSTable+Bloom)零依赖，格式可控
KV 缓存32路分片 sync.Map + TTL 最小堆O(1)读写，低锁竞争
向量索引自研纯 Go HNSW无 CGo，可交叉编译 ARM
关键词检索BM25 倒排索引 + 中英分词精确匹配，200行代码
HTTP 服务Go net/http 标准库零框架依赖
前端原生 HTML/CSS/JS + Canvas零 Node.js 依赖
AI 模型DeepSeek V4 Pro (可替换)OpenAI 兼容协议

设计原则：一个二进制文件，import 即用，零外部依赖。

项目结构

nebula-agent/
├── engine/                     ← 核心引擎（可嵌入 Go 包）
│   ├── storage/lsm/              LSM Tree (WAL+SkipList+SSTable+Bloom)
│   ├── storage/kv/               分片锁 KV + TTL
│   ├── storage/vector/           纯 Go HNSW 向量索引
│   ├── retrieval/                Embedder + 混合检索(RRF)
│   ├── index/                    倒排索引 + BM25
│   ├── manager/                  四种记忆 + 整合引擎
│   └── cache/                    LRU 缓存
├── analyzer/                   ← 代码分析（扫描·学习·注入）
├── api/rest/                   ← REST API + DeepSeek 对话
├── cmd/nebula-server/          ← 启动 Web 服务
├── cmd/nebula-code/            ← CLI 终端工具
├── web/                        ← 前端仪表盘（莱茵生命风格）
├── sdk/go/                     ← Go SDK
└── sdk/python/                 ← Python SDK

快速开始

Web 服务（带前端仪表盘）

cd nebula-agent
go run ./cmd/nebula-server --memory --port 8730
# 浏览器打开 http://localhost:8730

CLI 三步走

# 1. 扫描项目，学习代码特征
go run ./cmd/nebula-code ingest --dir .

# 2. 查看引擎学到了什么
go run ./cmd/nebula-code style
go run ./cmd/nebula-code gotchas

# 3. 生成 AI 上下文（复制粘贴到任何 AI 工具）
